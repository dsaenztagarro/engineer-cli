### Added

- **Target writes go queue-aware ‚Äî declare, adjust & retire on Progress survive a tunnel.** The Progress screen's three target verbs (`n` declare, `e` adjust, `x` retire) now route through `QueuedClient` like every timer/week write: drain-before-live, enqueue on transport failure, loud if even queueing fails ‚Äî an offline gesture never bounces. The queue's `IntentKind` gains additive, serde-tagged `target_create`/`target_adjust`/`target_retire` variants: a declare carries the whole `TargetCreate` body on the shared `"target"` stream and replays under its queued `Idempotency-Key` (keyed strictly dominates a plain re-send ‚Äî where targets-create is in ADR 0036's opt-in set the server dedupes a lost ack, and where it is not the header is ignored; a replay re-sends the identical body under the identical key, so a key-reuse conflict cannot arise); adjust/retire key on their lineage row (`"target:<id>"`) and replay plain. The replay honors the append-only lineage rule (engineer ADR 0026): an adjust that lands on a version closed while offline gets `target-version-closed` back with `live_target_id`, and **re-addresses the same hours to the live row instead of diverging** ‚Äî the gesture ("this many hours on this lineage") still lands; only a lineage with no live row left (fully retired meanwhile) is a genuine divergence and halts the drain. Live, the same coded conflict renders the shared soft "moved on ‚Äî re-fetching the live pace" copy and refetches, never a hard error. An offline declare renders a provisional `‚óî scope ¬∑ Nh/wk queued` line under the meters (the Week board's idiom) until it replays and a live refetch returns the real reading; offline adjust/retire confirm `¬∑ queued (offline) ‚Äî will sync`. - **The git-source connect flow ‚Äî trust-gated, honest about what it can't do (`c` from the Inbox).** The epic's last surface, and the terminal-native half of assisted-capture: the step that makes drafts *appear*. A new capture-source client (`src/api/capture.rs`) over the shipped `/api/v1/capture/sources` contract (dsaenztagarro/engineer#804, engineer PR #808, ADR 0035) powers both a TUI screen and a headless twin. On the **Connect screen** (`src/app/screens/connect.rs`, reached from the Inbox via `c` ‚Äî the design's footer key), a list of the capture sources (git / calendar) with their connect state, and three verbs behind a modal: `c` connects ‚Äî rendering the source's plain-language **trust statement verbatim *before* connecting** (`reads` / `never_reads` / `promise`, straight from the payload ‚Äî the CLI never invents its own wording), then a confirm (the git source takes no body; the calendar captures a feed URL first); `d` disconnects, behind a confirm that says what it means ‚Äî **disconnect turns the source off, it doesn't delete the drafts it already captured** (disconnect ‚âÝ delete); `s` enqueues a scan for a connected source. ## [0.8.0] - 2026-07-16

### Added

- **Timer divergence is a surfaced choice ‚Äî the reconcile panel, three resolutions, `engineer queue resolve`.** The honesty deliverable of the offline-write epic: when replay finds the server has moved on (a session started on the web, a stop the server refused), the diverged intent is no longer just a halted queue row ‚Äî it becomes a decision put in front of you, and **no path loses a segment silently**. In the TUI, the Timer screen surfaces a full-row danger reconcile panel (`Two sessions ‚Äî which is real?` / `The server refused this save`): the server's objection verbatim from the stored RFC 7807 payload, the local intent's identity (verb word, queued age, the local clock), two sides picked with `j/k` + `‚èé`, `b` for keep-both, Esc to defer (it reopens on the next poll while the divergence stands) ‚Äî and the header wears a ` diverged ` danger chip on every screen, the one loud state in the vocabulary. The resolutions are compositions of existing typed calls, one spelling for both surfaces (`queue::resolve`): **keep local** re-asserts your gesture (`start_timer(switch: true)` stops & saves the server session and yours takes over; a diverged *stop*'s minutes are written explicitly via `create_segment`), then the drain continues behind the choice; **take server** moves the local session's intents to a new `IntentState::Parked { reason }` ‚Äî kept in `queue.json` for review, visible as `parked` in `engineer queue`, excluded from replay and from every queued count, **never deleted**; **keep both** composes the local session from what is actually stored (a queued start's anchor, a queued stop's `local_elapsed_s`, else the folded clock at now) and writes it via `create_segment` while the server session stands. Headlessly, `engineer queue resolve <id> --keep=local|server|both` is the exact twin (plain + `--json`, refusals exit 1 and change nothing); the bare `engineer queue` read now names each waiting objection under the table with the resolve command spelled out (exit 4 as shipped); and every status surface tells the truth ‚Äî `diverged: bool` in `engineer timer --json`, ` diverged=1` in the plain status extras, `¬∑ diverged ‚Äî resolve in engineer queue` on the human read. Where the generic-conflict payload can't compose a resolution (no server segment id until the coded conflicts land, an unbound local session), the arm refuses with the reason and the intent stays diverged ‚Äî loud, kept, honest. (`offline-write.brief.md` job 4, ¬ßDiverged ¬∑ session elsewhere + ¬ßDiverged ¬∑ clock drift; EPIC #98, #106)

- **Reconnect UX in the TUI ‚Äî the queue drains visibly and quietly.** When the wire returns and the header poll replays queued writes, the moment is now *visible and calm* instead of silent. A one-line ambient transcript rides the shipped notify surface while intents land ‚Äî `back online ¬∑ replaying the queue‚Ä¶ start ¬∑ pause ¬∑ resume`, each verb word appended as the server acknowledges it ‚Äî and a clean drain that reconciled at least one write settles into a single `‚úì synced ‚Äî N queued writes reconciled` tile that auto-dismisses on its own (~4s, `Level::Success`'s TTL). A pass that replayed nothing shows nothing; a pass halted by a divergence shows no "synced" and lets the existing diverged markers stand (the loud reconcile panel is a later slice); and if another process holds the `replay.lock`, the drain skips and the TUI stays quiet ‚Äî no false transcript. The drain streams progress through the reducer via two new TEA actions (`ReplayProgress { word }` per acknowledged intent, `ReplayFinished(ReplayReport)` for the summary the tile reads), fed by a `QueuedClient::drain_reporting` that wraps the replay pass with a per-intent callback ‚Äî the `TimerProvisional`-style plumbing the offline verbs already use. The footer is unchanged; the ambient `‚ÜëN` count clears and the watch face's `‚óî` becomes `‚óè` as the next live poll lands. (`offline-write.brief.md` ¬ßReconcile ¬∑ replaying + ¬ßReconcile ¬∑ merged; EPIC #98, #105)

- **`engineer timer` write verbs work offline too ‚Äî the headless twin of the train case.** The one-shot CLI now matches the TUI: `start` / `stop` / `pause` / `resume` / `toggle` / `bind` / `discard` route through `QueuedClient`, so a tunnel no longer errors the write ‚Äî the gesture queues and the provisional outcome prints, marked `¬∑ queued (offline)` (`‚óè started  0:00:00  untitled ¬∑ queued (offline)`, `‚ñÝ saved 45m ¬∑ queued (offline)` ‚Äî no segment id, it's server-minted on replay). A bare `engineer timer start` enqueues an unnamed clock; `toggle` and `discard` read the effective (folded) local clock offline, so toggle flips the right way and `--force` fences against the elapsed the user actually sees. Two reads stay live because they can't be synthesized: a *query'd* `start`/`bind` needs the server's fuzzy match (`offline ‚Äî can't resolve "raft"; start bare or retry online`), and `stop --reclaim` needs the server's idle verdict (`offline ‚Äî reclaim needs the server's idle verdict; retry online`) ‚Äî both refuse clearly (exit 1) rather than guess. Exit codes are unchanged: a queued write still exits 0. (`offline-write.brief.md` job 1, the ¬ßC headless contract; EPIC #98, #104)

- **Control the timer entirely offline in the TUI ‚Äî every verb keeps working, marked queued.** The write half's headline case (the train in a tunnel): start / pause / resume / stop / bind / discard on the Timer screen ‚Äî and the `:timer` palette verbs ‚Äî no longer refuse the keystroke when the wire is down. Each acts on the local clock, enqueues a persisted intent, and renders provisionally: the watch face wears a `‚óî QUEUED` marker, the header cell gains a quiet ` ‚ÜëN` unsynced-writes count (accent, the shipped ` ~` stale idiom's family), and an offline stop shows the usual segment confirmation marked `‚óî segment queued` (no server id, no undo, "will sync when you reconnect"). Every timer write in `src/app/screens/timer.rs` now routes through `QueuedClient` ‚Äî which grew the remaining verbs (`start_timer`/`stop_timer`/`bind_timer`/`discard_timer`) alongside the shipped pause/resume, each drain-before-live then enqueue-on-`Transport`, synthesizing the stand-in from `timer_clock`. When the network returns the TUI reconciles on its own: the header poll drains the queue before every read, so queued writes replay in order without needing another keystroke. The read cache stays server-truth-only ‚Äî the folded/provisional clock is never written back ‚Äî and non-transport errors (auth, a server rejection) keep today's notify-tile semantics exactly. (`offline-write.brief.md` job 1; EPIC #98, #103)

- **The offline clock now advances ‚Äî and freezes, and resumes ‚Äî exactly like the server's.** Offline reads stop rendering a frozen snapshot: on a transport failure, `engineer timer` / `status` / `--short` now compose the cached server snapshot with any pending queued writes (`queue::fold_timer`) and compute elapsed with the server's own arithmetic ‚Äî `started_at` plus banked `paused_seconds`, plus the open paused span while paused (`src/timer_clock.rs`, which also gains the 4 AM study-day boundary mirror and the `apply_start`/`apply_stop` transitions the offline verbs will ride). So in a tunnel the status-bar clock keeps counting, a queued pause freezes it, a queued resume advances it again, and a queued stop reads `nothing running` ‚Äî all wearing the shipped stale/queued markers, and self-updating the moment the queue drains (the fold re-reads the queue on every read). The TUI header does the same: a failed poll renders the folded local clock with a ` ~` instead of silently extrapolating a clock the queue may have paused, and the live between-poll tick now uses the same arithmetic, so it can never drift from a stale poll. `timer-cache.json` stays server-truth-only ‚Äî the provisional folded clock is never written back.

- **Offline `timer pause`/`resume` ‚Äî the write queue becomes visible behavior.** The first slice of the offline *write* half (the read half shipped in v0.7.0): when the wire is down, `engineer timer pause`/`resume` (and `toggle`) no longer refuse the keystroke ‚Äî the gesture lands in a persisted local queue (`queue.json`, sibling to the read cache) and the output says so plainly (`‚Äñ paused at 47:12 ¬∑ queued (offline)`), with the reads wearing the marker until it syncs (`‚ÜëN` on `--short`, `queued=N` on `status`, `"queued"`/`"queue_depth"` in `--json`). When the network returns the queue drains itself ‚Äî strict FIFO, one intent at a time, each re-send carrying the per-intent `Idempotency-Key` minted at enqueue so a lost ack can never double-write ‚Äî triggered before any live timer write (a fresh write never jumps the queue) and after any successful headless read. A server rejection halts the drain loudly: that intent is marked `diverged` with the full RFC 7807 payload kept, and nothing later replays past the open choice. And `engineer queue` makes it observable headlessly: the bare read prints a `#  INTENT  TARGET  AGE  STATE` table (`queue empty` when calm), `sync` runs a drain now (`‚úì synced ‚Äî 2 replayed` / `3 still queued, offline`), `--json` emits `{ depth, oldest_age_s, diverged, intents: [...] }`, and the exit codes answer "does the queue need me?": `0` drained/empty ¬∑ `3` queued offline (not a failure ‚Äî cron must not page on a tunnel) ¬∑ `4` a divergence is waiting ¬∑ `5` replay failed. (The full local clock ‚Äî all timer verbs offline ‚Äî the reconnect transcript, and the divergence *resolve* flow are the next slices; `offline-write.brief.md`, EPIC #98.)

## [0.7.0] - 2026-07-15

### Added

- **Offline read-cache ‚Äî the headless timer never goes blank.** The status-bar reads (`engineer timer`, `status`, `--short`) now cache the last-known timer to the state dir on every success and fall back to it when the network is unreachable ‚Äî so a dropped connection (studying on a train) shows `‚óè 52:14 ~` or `‚Ä¶ ¬∑ offline (last known 2m ago)` instead of nothing. `--json` gains `"stale": true` + `"stale_age_s"`. Backed by a new automations client. (The TUI triage screen and the terminal-native git-source connect flow ‚Äî the latter blocked on a server source-registration contract ‚Äî are the Phase-2 follow-ups.) (`assisted-capture.brief.md`)

- **`engineer week` and `engineer plan` ‚Äî the week, planned vs done.** `engineer week [--week YYYY-Www]` prints the readout ‚Äî each plan item with its state (`done ‚úì` / `missed` / `planned` / `in progress`) and the summary (`planned 3 ¬∑ done 1 ¬∑ 4.2h logged`); `--json` for the full aggregate. `engineer plan '<title>' [--on YYYY-MM-DD] [--size <min>] [--kind K]` declares a plan item ‚Äî a `planned` activity on a day, so the plan and the actuals stay one ledger (no separate plan endpoint; `planned_on`/`target_duration_minutes` added to the activity create). - **`$EDITOR` for note bodies ‚Äî `Ctrl-E` in quick-capture.** Long-form prose now opens in your `$EDITOR` (or `$VISUAL`, else `vi`) the `git commit` way: `Ctrl-E` in the capture overlay suspends the TUI, drops you into the editor seeded with the current body, and folds the saved text back into the note on exit ‚Äî the alt-screen and raw mode restored around the child process. An empty buffer or an aborted editor keeps the original (empty-buffer-cancels). The one-line quick-capture path (type then `Ctrl-S`) stays as-is for a fast thought; `$EDITOR` is for when the thought grows past a line. (`notes.brief.md`; `cross-cutting.brief.md` ¬ßE)

Save is explicit (`Ctrl-S`, since Enter is a newline in the editor); **capture is sacred** ‚Äî an accidental `Esc` on a non-empty draft only warns (`Esc` again to discard), so input is never lost. On save the note POSTs to `/api/v1/notes`, adding a book + page anchor when both are set. A new **Notes browser** (`n` from Home or the `<Space>` leader, or `:notes`) lists notes with a one-line anchor read-back for anchored notes (`SICP ¬∑ ch 3 ¬∑ p.142`) next to loose thoughts, `/` searches the server, `j`/`k` navigates, `‚Üµ` opens a full-content detail read, `a` archives/unarchives, `t` reveals archived notes (dimmed in place), and `e` reopens the same overlay pre-filled to edit (PATCH).

- **Timer screen + persistent header cell.** A new `Timer` screen owns the daily-loop timekeeping moments: start a blank clock ("name it later"), bind a running timer to work by searching `/api/v1/timer/candidates` or minting a new activity from a typed title, pause/resume, and stop ‚Äî which shows the written segment (minutes ‚Üí activity) so the ledger is trusted. On top, a fixed-width **header cell on every screen** renders the live timer per the web pill contract: an accent `‚óè mm:ss` (widening to `h:mm:ss`) when running, an amber `‚Äñ mm:ss` when paused, and nothing when absent ‚Äî never the activity title. The snapshot lives in app state and is polled every 15s from `GET /api/v1/timer`, with the displayed elapsed ticked locally between polls. Reachable from Home with `t` (or the `<Space>` leader `t`, or `:timer`); on-screen keys are `s` start ¬∑ `/` bind ¬∑ `p` pause/resume ¬∑ `x` stop ¬∑ `d` discard.

- **Progress screen (`engineer pace` meters).** A new read-only screen renders the weekly pace aggregate from `GET /api/v1/progress` as one ASCII meter per target, behind-first: a block-bar with a now-tick (where the week expects you to be), `actual/target` hours, the signed delta, and the pace state (`behind` amber / `on pace` green / `met`). A week-header line (`2026-W27 ¬∑ sat ¬∑ day N of 7 ¬∑ now = NN%`), a behind-total footer naming the largest gap, and a compact kind-mix line round it out. Reachable from Home with `p` (or the `<Space>` leader `3`/`p`); step weeks with `[` / `]` and return to the current week with `t`. Pace is derived from segments at read time, so nothing is stored client-side.

## [0.2.0] - 2026-06-21

### Changed

- **Renamed the project from `engineer-tui` to `engineer-cli`.** The package name, OS keyring service, XDG config/state directories (`~/.config/engineer-cli/`, `~/.local/state/engineer-cli/`), log filename prefix, tracing namespace (`engineer_cli::api`), log env var (`ENGINEER_CLI_LOG`), and the CIMD `client_id` document path (`engineer-cli.json`) all moved to the new name. The installed binary remains `engineer`. **Upgrade note:** because the keyring service name changed, existing installs must re-run `engineer login`; any `~/.config/engineer-tui/config.toml` should be moved to `~/.config/engineer-cli/config.toml`.

### Added

- **Prebuilt binaries and Homebrew distribution via [cargo-dist](https://opensource.axo.dev/cargo-dist/).** Tagged releases now cross-build macOS (arm64/x86_64) and Linux (arm64/x86_64) binaries, publish them to GitHub Releases with a shell installer, and push an `engineer` formula to the [`dsaenztagarro/homebrew-tap`](https://github.com/dsaenztagarro/homebrew-tap) tap. Install with `brew install dsaenztagarro/tap/engineer`.

## [0.1.0] - 2026-06-19

### Added

- Initial release: a neovim-flavored terminal client for [Engineer](https://github.com/dsaenztagarro/engineer) to log activities and track reading progress, authenticating against [Identity](https://github.com/dsaenztagarro/identity) over OAuth2 Authorization Code + PKCE (RFC 8252).
- **Zero-friction authentication via an OAuth Client ID Metadata Document (CIMD).** The client identity is the URL of a metadata document served by the Engineer app; Identity fetches it server-side. The `client_id` is derived from `api_url`, so no manual client registration or pasted credentials are needed.
- **In-TUI sign-in screen.** Launching without a stored refresh token lands on a "Sign in" screen; pressing Enter runs the browser OAuth flow and drops into the authenticated TUI. The refresh token is stored in the OS keyring; subsequent launches go straight in. The `login`, `logout`, and `whoami` subcommands remain for scripting.
- **Explicit environment selection** via the `--env` flag and `ENGINEER_ENV` variable (`production` default, or `development` for localhost), with built-in URL presets so a fresh run needs no config file. Layered configuration: environment preset < `~/.config/engineer-cli/config.toml` (XDG-honored on all platforms, including macOS) < `ENGINEER_*` env vars.
- **GitHub Actions CI** running `cargo test` on pushes to `master` and on pull requests.
