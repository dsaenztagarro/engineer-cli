WEEK-PLANNING · engineer-cli · declare the week & read planned-vs-done
The plan and the retro are two faces of one ISO week. On Monday you declare what the week is for — a short list of intents, a keystroke each. On Friday you read the honest planned-vs-done — SICP done · systems 2 of 3 · that paper never happened — and write why in $EDITOR, the way you write a commit message. A readout and a one-liner — not a planning canvas.
This canvas reads top to bottom: the board (plan ⇄ retro) → the readout dissected + the empty week → declaring the week → the $EDITOR reflection → the headless twins & the API. The actuals half is derived from GET /api/v1/weeks/:iso_week; the plan items are real activity rows written through the activities API.

4 THE REFLECTION — $EDITOR for prose, never a textarea in ratatui WRITE BLOCKED the render ships in phase 2; the persist waits on a v1 route
THE HANDOFF · press i on the retro → suspend the TUI → $EDITOR → read back
week note — the git-commit pattern
┌─ ratatui · retro ──────────────────────────────────┐ │ 3 done · 1 partial · 1 untouched │ │ reflection [ i ] write why ▾ │ └────────────────────────────────────────────────────┘ │ i ▼ nvim /tmp/engineer-week-2026-W28.md # suspend TUI, spawn │ :wq save # read the temp file back │ empty buffer # → cancels, no write ▼ ✓ reflection attached · persist pending a v1 route
The retro readout is a screen; the retro writing is a hand-off. Same convention as note capture — spawn $EDITOR on a temp file, save-and-quit persists, an empty buffer cancels. One stored line per week, never a rich editor rebuilt on a character grid.

5 HEADLESS TWINS & THE API — every read the board shows is also a one-shot the retro is a --json readout & a piped line · the plan-declare a verb

bash — the retro, piped
$ engineer week --json { "iso_week":"2026-W28", "planned_minutes":840, "done_minutes":740, "tally":{ "done":3, "partial":1, "untouched":1 }, "items":[ { "title":"SICP…", "kind":"read", "planned_m":180, "logged_m":190, "verdict":"done" } ], "note":"The timer epic landed…" } $ engineer week # stable, field-ordered plain line 2026-W28 3/1/1 740/840 "SICP done · systems 2 of 3 · Raft paper never happened" └─ iso ──┘ └d/p/u┘ └don/pln┘ └───────────── readout ─────────────┘ $ engineer week --short # the status-bar reduction W28 3·1·1 88%
Colour is gated on stdout().is_terminal() AND NO_COLOR unset — a pipe gets clean text. The plain line's column order never changes and uses - placeholders, so awk/cut don't break. The read twins are unblocked today; only the note write waits on §4's route.
The API it consumes
GET/api/v1/weeks/:iso_week — the derived aggregate: plan, actuals, pace fold, note. Read-only.
POST/api/v1/activities — declare a plan item (planned_on = the week).
PATCH/api/v1/activities/:id — adjust · :id/complete closes it done.
