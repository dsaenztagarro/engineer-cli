TIMER · engineer-cli
The timer, watch-face simple. One glorious number when you want it; a single status glyph when you don't. The instrument rail folds away with i — collapsed is the calm face, expanded is the cockpit. Every richer moment — idle reclaim, focus rhythm, the segment audit — is a full-row list you drive with j/k and ↵. Nothing here needs a mouse; everything here also runs headless.
Every state has all three forms: a face (full screen), a status-line segment (any screen), and a one-shot twin (piped). Durations in the copy — 50m work, 10m break, ×4, the idle threshold, the audit fences — are your settings, never constants.

Nothing is written until you choose; Esc defers — the clock stays frozen and this screen returns on your next keypress. Idle time is never logged as a segment. Presence = keystrokes in this TUI; a detached or closed TUI is absent — no screenshots, no global keylogging, ever. This screen appears on the next interaction (or at launch) once the idle threshold — yours, from settings — has passed.

Rows map 1:1 to the server's reclaim verbs — trim / keep / stop — plus the discard escape (the whole-timer verb, confirm per §Saved & undo). The default selection is your default reclaim action setting.

SAVED · UNDO · DISCARD CONFIRM (the footer tile moments)
nothing modal — the footer speaks, the screen stays
AFTER s — SAVED (notify tile, success · auto-expires)
✓ 52m saved to Read DDIA ch.7 — Transactions · today now 1h 30m   u  undo · 30s
u deletes the just-written segment — the exact inverse, powered by segment delete. After the tile expires, the segment is a normal row in the activity's history (and in §Segment audit if flagged).
AFTER d — DISCARD ASKS FIRST (warn tile, fills the row)
⚠ discard 41m? nothing will be written d  confirm  ·   Esc  cancel
Confirmation only past ~2 minutes — a mis-tap discards instantly, real work asks twice. The same confirm guards ✗ Discard on §Idle reclaim and engineer timer discard (headless: --force skips it).
Both are the shipped footer idioms — a notify tile in the hint row, never a modal. Undo has no server "undo": it's one segment delete, so it's honest about what it can reverse (the last save only, while the tile shows).

MODE SWITCH · ON THE RUNNING TIMER
m (or :focus / :stopwatch) switches a running timer in place — stopwatch → focus carries elapsed work into interval 1; focus → stopwatch drops the rhythm, keeps the clock. Nothing stops, nothing is written.
Away from the hero, the offer is the status-line ◆ break? segment (breathing, §Status line) plus a one-shot notify tile — g t jumps here to decide. n mid-interval short-circuits offer 1: it banks the interval and arms the next.

EXIT CODES — "is the clock counting?"
0 counting — running, focus work, or over  ·  1 nothing running  ·  3 idle — reclaim pending  ·  4 not counting — paused or on a break
Scripts branch on the code for counting/not; anything finer reads the first word. over keeps exit 0 — it is still work; the word and "over": true carry the nuance.
