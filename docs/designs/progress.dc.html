Decision record — targets are append-only versions (ADR 0026)
Retire ≠ deleteThere is deliberately no DELETE. PATCH :retire closes a lineage while keeping its history so past weeks still read it — the verb is retire, and the confirm says “keeps history,” never “delete.”
Adjust mints a successorPATCH :id returns the live row, whose id can differ; a closed version id is a 422. So the CLI addresses a lineage by axis + scope, re-reads after an adjust, and surfaces the 422 as “this target moved on,” not a hard error.
One clock · derivedWeek attribution uses the 4 AM study-day boundary and Monday-first ISO week (YYYY-Www); a meter and a status string agree with the web to the minute. Pace and rollups are read-through from GET /api/v1/progress — the only write this module owns is the target itself.
